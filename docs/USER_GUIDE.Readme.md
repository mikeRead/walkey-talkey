# USER_GUIDE

## Useful Information for Humans

- `USER_GUIDE.md` is the operator-facing guide for setup, Wi-Fi portal access, save/reset behavior, and recovery steps.
- Keep it practical and task-oriented.
- It should explain the current shipped credentials, URL, startup delay, and the meaning of `STORAGE_FAILED` details in plain language.
- It should stay aligned with `README.md`, `config/README.md`, and `docs/mode-system-reference.md`.

## Useful Information for AI

- Prefer updating `USER_GUIDE.md` when portal credentials, startup timing, recovery flow, or troubleshooting behavior changes.
- Keep the guide end-user friendly; avoid deep internal implementation detail unless it directly affects recovery.
- If the reset target or storage diagnostics change, update this file in the same pass.
